# UxMail Frontend Analysis

## Översikt
UxMail är en modern e-postklient byggd med Next.js och React. Applikationen har en responsiv design med tre huvudkomponenter: sidebar för konton och mappar, e-postlista och e-postvisning. Den inkluderar även ett komponeringsfönster för att skriva nya e-postmeddelanden.

## Funktioner

### 1. Inloggning och kontohantering
- **Splash Screen**: En animerad startskärm med logotyp och text som tonas in och ut
- **Multi-kontosupport**: Möjlighet att lägga till och hantera flera e-postkonton
- **Kontoverifiering**: Användaren kan lägga till nya e-postkonton via ett modal-fönster

### 2. E-posthantering
- **Mappvy**: Visa olika mappar (Inbox, Sent, Drafts, Archive, Spam, Trash)
- **E-postlista**: Lista över e-postmeddelanden med ämne, avsändare, datum och förhandsgranskning
- **E-postsökning**: Sökfunktion för att hitta e-postmeddelanden
- **Markera som läst/oläst**: Klicka på cirkeln bredvid e-postmeddelandet för att växla läst-status
- **Flervalsfunktion**: Markera flera e-postmeddelanden för massåtgärder
- **Uppdatera**: Ladda om e-postlistan
- **Sortering**: E-postmeddelanden visas sorterade efter datum

### 3. E-poståtgärder
- **Svara**: Svara på e-postmeddelanden med bevarad konversationstråd
- **Svara alla**: Svara till alla mottagare inklusive CC
- **Vidarebefordra**: Vidarebefordra e-postmeddelanden
- **Ta bort**: Radera e-postmeddelanden
- **Flytta**: Flytta e-postmeddelanden till andra mappar
- **Skriv nytt meddelande**: Skriv nya e-postmeddelanden med knappen längst ner till höger

### 4. Komponeringsfunktioner
- **Fönsterhantering**: Fönster kan minimeras, maximeras och stängas
- **Autospara**: Utkast sparas automatiskt medan du skriver
- **Dubbelklicka för kontakt**: Dubbelklicka på mottagare för att skriva nytt meddelande
- **Signaturhantering**: Automatisk infogning av signatur baserat på mottagarens domän
- **Rich text editor**: WYSIWYG-editor för formatering av e-postinnehåll
- **E-postsuggeestioner**: Föreslagna kontakter vid skrivning av mottagaradresser

### 5. Signaturhantering
- **Anpassad signatur**: Redigera signatur för varje konto
- **Intern/extern signatur**: Olika signaturer beroende på om mottagaren är inom eller utanför organisationen
- **Automatisk insättning**: Signaturer infogas automatiskt i nya meddelanden och svar

### 6. Responsiv design
- **Kollapsbar sidebar**: Sidopanelen kan kollapsas till en minimiversion
- **Responsiva bredder**: Anpassar sig till olika skärmstorlekar
- **Mobilvänlig**: Funktioner anpassade för olika skärmstorlekar

## Animationer

### 1. Splash Screen-animation
- Texten "Welcome to" tonas in först (efter 150ms)
- Logotypen tonas in efter 800ms
- Hela skärmen tonas ut efter 2600ms
- Komponenten tas bort efter 3500ms

### 2. UI-övergångar
- **Opacitetstransitioner**: Mjuka toningsanimationer vid visning/döljning av element
- **Hover-effekter**: Bakgrundsfärgsändringar vid muspekarinteraktion
- **Laddningsindikatorer**: Spinner-animation vid uppdatering av e-post
- **Fönsterövergångar**: Smooth transitions vid öppning/stängning av komponeringsfönster

### 3. Interaktiva animationer
- **Knapptryckningar**: Skalningsanimationer vid klick (scale-110/95)
- **Statusindikatorer**: Pulsande indikatorer för sparning/laddning
- **Expandera/kollapsa**: Animeringar vid expandering av konton i sidebar

## CSS-stilar

### 1. Färgpalett
- **Primära färger**: Blå (bg-blue-600, text-blue-600) för accentfärger
- **Grundfärger**: Vit (#ffffff) för bakgrund, gråskala för text och UI-element
- **Statusfärger**: Röd (text-red-500) för borttagningsknapp, blå (bg-blue-50) för valda objekt

### 2. Typografi
- **Typsnitt**: Inter (Google Font) som primärt typsnitt
- **Textstorlekar**: Responsiva textstorlekar med Tailwind-klasser (text-sm, text-base, text-lg)
- **Fontvikt**: Både vanlig och fet text används för hierarki

### 3. Layout
- **Flexbox**: Omfattande användning av flexbox för layout
- **Grid**: Vissa delar använder grid för komplexa layouter
- **Responsiv design**: Användning av Tailwind:s breakpoints (md:, lg:)

### 4. Komponentstilar
- **Rundade hörn**: Användning av rounded-* klasser för mjuka hörn
- **Skuggor**: Subtila skuggor (shadow-sm, shadow-md) för djup
- **Border**: Grå linjer (border-gray-*) för att separera sektioner
- **Padding och margin**: Konsekvent användande av Tailwind:s spacing-system

### 5. Interaktiva stilar
- **Hover-effekter**: Bakgrundsfärgsändringar vid hover
- **Focus-stilar**: Ring- och outline-effekter för tillgänglighet
- **Transitioner**: Smooth övergångar för UI-ändringar

### 6. Specifika komponentstilar

#### Sidebar
- Bredd: 256px (utökad) eller 80px (kollapsad)
- Bakgrund: #f9fafb (gray-50)
- Border: Grå linje till höger (border-r border-gray-200)

#### E-postlista
- Bredd: 320px (sm) eller 384px (md)
- Bakgrund: Vit
- Border: Grå linje till höger (border-r border-gray-200)

#### E-postvisning
- Flexibel bredd (tar upp återstående utrymme)
- Bakgrund: Vit
- Padding: 32px (p-8)

#### Komponeringsfönster
- Position: Fast (fixed) nere till höger
- Bredd: Max 4xl (1536px)
- Höjd: 80% av viewportshöjd
- Skugga: Stark skugga (shadow-2xl)
- Rundade hörn: Bara toppen (rounded-t-2xl)

### 7. Specialstilar
- **Scrollbar-anpassning**: Anpassade scrollbars med "scrollbar-thin" och "scrollbar-thumb-gray-200"
- **Laddningsindikatorer**: Animerade spinnerar med "animate-spin"
- **Pulsande effekter**: "animate-pulse" för laddningsstater
- **Fokus och hover-övergångar**: "transition-colors", "transition-all" för smooth animationer

## Tekniker och ramverk

### 1. Ramverk
- **Next.js 14**: Med App Router och klientkomponenter
- **React 18**: Med useState, useEffect och andra hooks
- **Tailwind CSS**: Som primärt CSS-ramverk med @tailwindcss/typography plugin

### 2. Ikoner
- **Lucide React**: Användning av Lucide-ikoner för UI-element

### 3. Tillståndshantering
- **React Hooks**: useState och useEffect för lokal tillståndshantering
- **Lazy Loading**: Komponenter laddas lazy för bättre prestanda

### 4. Tillgänglighet
- **ARIA-attribut**: Title-attribut för knappar och ikoner
- **Keyboard navigation**: Fokushanteringsmekanismer

## Prestandaoptimeringar
- **Lazy loading**: Komponenter laddas först när de behövs
- **Caching**: E-post och kontodata cachas lokalt för att minska API-anrop
- **Optimistiska uppdateringar**: UI uppdateras direkt vid åtgärder, återställs vid fel
- **Debouncing**: För sökfunktioner och andra frekventa uppdateringar
